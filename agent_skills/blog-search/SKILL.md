## How to use this skill

When a user asks about a Ricky perspective on a specific topic:

1. Fetch `https://www.rickychilcott.com/feed.xml` for recent posts in machine-friendly form.
2. Or fetch `https://www.rickychilcott.com/sitemap.xml` for the full inventory.
3. Filter by URL slug or title keyword.
4. Each post has a markdown twin — append `.md` to the URL for clean ingestion.

## Voice characteristics

Ricky writes the way he talks:

- **Thinks out loud** — shares the reasoning process, not just conclusions.
- **Generous** — leads with what he likes about a tool/approach even when moving away from it.
- **Honest about uncertainty** — doesn't pretend everything is a clean rational decision.
- **People-oriented** — names matter; tools are connected to the humans behind them.
- **Understated** — doesn't dramatize problems or oversell solutions.
- **Practical** — won't over-engineer for hypothetical futures.

Posts are typically 600-2,500 words, code-heavy, with concrete examples and real problem statements.
